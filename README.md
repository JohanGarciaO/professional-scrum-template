# Professional Scrum Template

> A technology-agnostic repository template that teaches software engineering principles and provides a professional Scrum-based foundation for starting software projects.

---

## Overview

The **Professional Scrum Template** is more than a repository template.

It is an educational engineering foundation that demonstrates how professional software projects can be planned, documented, and managed using Scrum and complementary software engineering practices.

Instead of providing a ready-made project structure or prescribing specific technologies, the template teaches the principles behind each engineering artifact, allowing teams to understand not only **what** to create, but also **why** it exists.

---

## Philosophy

This template is based on one simple principle:

> **Engineering comes before implementation.**

Before writing code, a project should define:

- What problem it solves.
- Who its users are.
- What value it delivers.
- How work will be organized.
- How quality will be measured.

Technology changes.

A good engineering process does not.

---

## Design Principles

The template follows a few fundamental principles:

- Teach engineering principles before implementation details.
- Remain technology-agnostic.
- Provide reusable documentation instead of project-specific artifacts.
- Prefer established industry standards over custom conventions.
- Keep the template minimal, but complete.

> **The template defines principles. The examples demonstrate one possible implementation.**

Core documents describe universal engineering concepts that apply to any software project.

Whenever concrete examples are necessary, they illustrate only one possible implementation and should never be interpreted as mandatory project requirements.

---

## What this template provides

- Scrum-oriented project structure.
- Documentation templates.
- Product management artifacts.
- Definition of Ready.
- Definition of Done.
- GitHub Issues templates.
- Pull Request template.
- ADR (Architecture Decision Records).
- Recommended GitHub Projects workflow.

---

## What this template does NOT provide

This template intentionally does **not** define:

- Programming language.
- Framework.
- Database.
- Architecture.
- Infrastructure.
- CI/CD pipeline.
- Source code.

Those decisions belong to the project that uses this template.

## Getting Started

### 1. Create a repository

Click on **Use this template** to create your own repository.

### 2. Read the documentation

Go through the documents inside the `docs` directory in order.

### 3. Define your product

Start by filling in:

- Product Vision
- Product Goal
- Roadmap

### 4. Configure GitHub Projects

Create your Scrum board following the recommended workflow.

### 5. Build your Product Backlog

Break your product into features and user stories.

### 6. Start Sprint 1

Plan your first Sprint and begin development.
